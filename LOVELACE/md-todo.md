# TODO: Клиентская часть Lovelace

## Исходная постановка

Теперь с клиентской частью сделай следующее:

1. Раздели на две страницы.
2. Мы попадаем на каждую из страниц в зависимости от того, какой пароль ввели (в Виженере есть похожая механика): `010` ведёт на текущую, а `101` — на новую.
3. Новая страница должна, используя все те же механики, уйти от ввода-вывода и арифметики вообще. Вместо этого сосредоточиться на перемещении исполнителя по двумерному полю. Допускается перемещение, перекраска полей, анализ состояния поля (например, числовая температура клетки), старт и финиш. В общем, классический «Исполнитель Робот».
4. Все остальные механики должны быть идентичны и синхронизированы по механикам (не по контенту) между версиями.

## Расширенный промпт для реализации

Сделай рефакторинг раздела `LOVELACE` в двух клиентских вариантах:

1. Создай две отдельные страницы внутри раздела:
   - текущая версия (существующая логика);
   - новая версия «Исполнитель Робот» (двумерное поле).
2. На входной странице раздела реализуй развилку по паролю:
   - пароль `010` открывает текущую страницу;
   - пароль `101` открывает новую страницу.
3. Для новой страницы:
   - убрать акцент на ввод/вывод и арифметику в заданиях;
   - сделать основной контекст: робот на сетке, перемещения, изменения состояния клеток, анализ состояния клетки, старт/финиш;
   - сохранить общий стиль взаимодействия, знакомый ученику по текущей версии.
4. Механики должны оставаться синхронными между двумя версиями:
   - одинаковые правила выполнения командного движка там, где это применимо;
   - одинаковые ограничения (валидация, ошибки, лимиты, сценарии запуска);
   - различаться должен контент/сценарии, а не базовая инженерная логика.
5. Не смешивать клиентские страницы с `dev`-контентом и не ослаблять существующую защиту разделов.

## Критерии приёмки

1. В разделе `LOVELACE` реально существуют две рабочие клиентские страницы.
2. Развилка по паролю работает строго по значениям `010` и `101`.
3. Новая страница реализует формат «Исполнитель Робот» (поле, состояние клеток, старт/финиш, действия робота).
4. Базовые механики редактора/движка между версиями синхронизированы.
5. Никакие открытые пароли/кодовые фразы не появляются в исходниках страниц.
